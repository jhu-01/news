# 📅 1주차 개발 체크리스트: 기초 설계 및 그리드 뷰 구현

## 프로젝트 기초 설정
- [x] **기술 스택 초기화**: Vite를 사용하여 React + TypeScript 환경을 구축하고, 디자인 시스템의 컬러 토큰 및 타이포그래피를 CSS 변수(`:root`)로 등록합니다.
- [x] **전체 레이아웃 구성**: 1280px 고정 너비를 가진 메인 컨테이너와 헤더, 티커, 메인 컨텐츠 영역의 위치(Gutter 175px)를 잡습니다.
- [x] **디렉토리 구조 설계**: `components/`, `hooks/`, `styles/`, `utils/` 등 도메인별 폴더 구조를 생성하고 `GEMINI.md` 가이드라인을 배치합니다.

## 헤더 및 티커 구현
- [x] **상단 헤더(Header)**: 로고 아이콘, "뉴스스탠드" 타이틀, 현재 날짜(연/월/일/요일)를 포함한 상단 영역을 구현합니다.
- [x] **뉴스 티커(Ticker) UI**: 배경색(#F5F7F9)과 1px 보더를 가진 두 개의 독립적인 롤링 라인을 배치합니다.
- [x] **티커 로직**: 3.2초마다 다음 기사가 나타나는 애니메이션(0.55s 크로스페이드)을 구현하고, 호버 시 롤링이 멈추는 기능을 추가합니다.

## 언론사 그리드(Grid) 구현
 - [x] **언론사 워드마크(Wordmark) 컴포넌트**: 이미지가 아닌 텍스트 조합으로 각 언론사 로고를 렌더링하는 범용 컴포넌트를 개발합니다.
 - [x] **6x4 그리드 레이아웃**: CSS Grid를 사용하여 24개의 셀(약 154x96px)을 생성하고, 셀 사이의 1px 경계선을 구현합니다.
- [x] **셀 호버(Hover) 상태**: 셀에 마우스를 올렸을 때 배경색 변경(#F5F7F9) 및 '+ 구독하기' 버튼(Pill)이 노출되는 인터랙션을 구현합니다.
- [x] **그리드 페이지네이션**: 72개 이상의 언론사를 처리할 수 있도록 좌우 셰브론 버튼을 통한 페이지 전환(1/3 페이지 등) 로직을 추가합니다.

# 📅 2주차 개발 체크리스트: 전역 상태 관리 및 리스트 뷰 구현

## 설계 및 검증 체계 (미션 필수 사항)
- [x] **상태 흐름 설계**: AI와 협업하여 구독(Subscription) 및 뷰 전환(Grid/List)에 대한 전역 상태 흐름도를 작성하고 승인받습니다.
- [x] **컴포넌트 계층 점검**: 리스트 뷰의 복잡한 데이터 전달 구조를 사전에 정의하고 props drilling을 최소화합니다.
- [x] **엣지 케이스 도출**: '구독 언론사가 0명일 때', '6초 타이머 도중 탭 클릭' 등 UX 엣지 케이스를 정의하고 검증 방법을 수립합니다.

## 전역 상태 및 구독 시스템
- [x] **SubscriptionContext**: Context API를 사용하여 구독한 언론사 ID 목록을 전역에서 관리합니다.
- [x] **Persistence**: `localStorage`를 연동하여 브라우저를 새로고침해도 구독 정보가 유지되도록 구현합니다.
- [x] **뷰 모드 전환**: 상단 탭(전체 언론사/내가 구독한 언론사) 및 우측 보기 설정(그리드/리스트) 연동 로직을 구현합니다.

## 리스트 뷰(List View) 구현
- [ ] **카테고리 탭 메뉴**: 6개 카테고리 탭과 각 카테고리 내 현재 페이지 정보를 표시하는 UI를 구현합니다.
- [ ] **6초 자동 롤링 (useTimer)**: `setInterval`을 활용하여 6초마다 다음 언론사/기사로 넘어가는 커스텀 훅을 개발합니다.
- [ ] **프로그레스 바 애니메이션**: 현재 활성 탭에 `#4362D0` 색상의 진행 표시줄이 6초 동안 채워지는 효과를 구현합니다.
- [ ] **기사 컨텐츠 영역**: 언론사 정보, 대표 기사(이미지 포함), 서브 기사 리스트를 `pressData` 기반으로 렌더링합니다.
