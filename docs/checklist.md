# 📅 1주차 개발 체크리스트: 기초 설계 및 그리드 뷰 구현

## 프로젝트 기초 설정
- [x] **기술 스택 초기화**: Vite를 사용하여 React + TypeScript 환경을 구축하고, 디자인 시스템의 컬러 토큰 및 타이포그래피를 CSS 변수(`:root`)로 등록합니다.
- [x] **전체 레이아웃 구성**: 1280px 고정 너비를 가진 메인 컨테이너와 헤더, 티커, 메인 컨텐츠 영역의 위치(Gutter 175px)를 잡습니다.
- [x] **디렉토리 구조 설계**: `components/`, `hooks/`, `styles/`, `utils/` 등 도메인별 폴더 구조를 생성하고 `GEMINI.md` 가이드라인을 배치합니다.

## 헤더 및 티커 구현
- [x] **상단 헤더(Header)**: 로고 아이콘, "뉴스스탠드" 타이틀, 현재 날짜(연/월/일/요일)를 포함한 상단 영역을 구현합니다.
- [x] **뉴스 티커(Ticker) UI**: 배경색(#F5F7F9)과 1px 보더를 가진 두 개의 독립적인 롤링 라인을 배치합니다.
- [x] **티커 로직**: 3.2초마다 다음 기사가 나타나는 애니메이션(0.55s 크로스페이드)을 구현하고, 호버 시 롤링이 멈추는 기능을 추가합니다.

## 언론사 그리드(Grid) 구현
 - [x] **언론사 워드마크(Wordmark) 컴포넌트**: 이미지가 아닌 텍스트 조합으로 각 언론사 로고를 렌더링하는 범용 컴포넌트를 개발합니다.
 - [x] **6x4 그리드 레이아웃**: CSS Grid를 사용하여 24개의 셀(약 154x96px)을 생성하고, 셀 사이의 1px 경계선을 구현합니다.
- [ ] **셀 호버(Hover) 상태**: 셀에 마우스를 올렸을 때 배경색 변경(#F5F7F9) 및 '+ 구독하기' 버튼(Pill)이 노출되는 인터랙션을 구현합니다.
- [ ] **그리드 페이지네이션**: 72개 이상의 언론사를 처리할 수 있도록 좌우 셰브론 버튼을 통한 페이지 전환(1/3 페이지 등) 로직을 추가합니다.

## 개발 Workflow (매 항목 반복)
1. **설계**: 기능을 구현하기 전 컴포넌트 구조와 상태 관리를 먼저 정의한다.
2. **구현**: AI에게 구체적인 프롬프트를 전달하여 코드를 생성한다.
3. **리뷰**: AI가 작성한 코드를 한 줄씩 읽으며 의도를 파악하고 질문한다.
4. **검증**: 브라우저에서 디자인 가이드와 일치하는지 확인한다.
5. **커밋**: `feat: #번호 기능명` 형식으로 커밋 로그를 남긴다.
